Parse vs Explicit Casting(명시적 캐스팅)

1. 'Parse'를 사용한 타입 캐스팅:
    'Parse'는 문자열을 해당하는 데이터 타입으로 변환하는 메서드이다. 주로 문자열을 숫자로 변환할 때 사용한다.
    이는 문자열을 분석하여 그에 해당하는 데이터 타입으로 변환하는 과정을 거친다.

    예를 들어, 문자열을 정수로 변환하기 위해 'int.Parse' 메서드를 사용할 수 있다.
    string str = "123";
    int number = int.Parse(str);  // 문자열을 int로 변환

2. 명시적 캐스팅(Explicit Casting):
   명시적 캐스팅은 큰 크기의 데이터 타입을 작은 크기의 데이터 타입으로 변환하는 것이다.
   이는 데이터의 손실이 발생할 수 있는 경우에 사용되며, 개발자가 명시적으로 변환을 지정해야한다.

   예를 들어, 실수를 정수로 변환하기 위해 명시적 캐스팅을 사용할 수 있다.
   double doubleValue = 10.5;
   int intValue = (int)doubleValue; // 실수를 정수로 변환

이 둘의 차이점:
  - 'Parse'를 사용한 타입 캐스팅은 주로 문자열을 해당하는 데이터 타입으로 변환하는 것에 사용된다. 
     데이터 타입간의 변환을 수행하는 것이 아니라 문자열을 해당 데이터 타입으로 변환한다.
  - 명시적 캐스팅은 큰 데이터 타입에서 작은 데이터 타입으로의 변환을 수행하며, 개발자가 변환을 명시적으로 지정해야한다.
    데이터의 손실이 발생할 수 있는 경우에 사용한다.

  따라서, 'Parse' 를 사용한 타입 캐스팅은 문자열을 데이터 타입으로 변환하는 것에 사용되고,
  명시적 캐스팅은 데이터 타입 간의 변환을 수행하는 것에 사용된다.
